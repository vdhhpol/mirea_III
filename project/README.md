# Итоговый проект по курсу «Инженерия Искусственного Интеллекта»

В этой папке находится итоговый мини-проект по курсу.  
Проект должен демонстрировать применение методов и инструментов инженерии ИИ: работу с данными, модели, пайплайны, сервис, эксперименты и (по возможности) воспроизводимость.

## 1. Паспорт проекта

- **Название проекта:** Кредитный скоринг: оценка риска невозврата займа
- **Автор:** Малышкина Варвара Александровна
- **Группа:** КВБО-21-24
- **Контакт:** @vdhhpol

- **Краткое описание (2-4 предложения):**  Проект реализует сервис кредитного скоринга на основе анкетных данных заёмщика. Используются открытые датасеты (Home Credit Default Risk). Сравниваются логистическая регрессия, LightGBM и простая нейросеть (MLP). Финальная модель упакована в REST API (FastAPI) с логированием, метриками, health-check и запуском в Docker.

## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - `model.py` – загрузка артефактов и функция `predict`;
  - `app.py` – FastAPI-приложение;
  - `config.py` – конфигурация из переменных окружения.
- `data/` – данные (в репозитории только примеры, полные данные не хранятся).
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

Если структура проекта отличается от базовой, опишите это здесь.

## 4. Как запустить проект

Опишите, как именно воспроизвести основные сценарии использования проекта.

Примеры (заполните под свой случай):

### 4.1. Запуск обучения модели

Обучение и эксперименты проводятся в **Jupyter Notebook** в папке `notebooks/`.  
Основные ноутбуки:

- `01_EDA_and_baseline.ipynb` – первичный анализ и baseline-модели.
- `02_Preprocessing_and_FeatureEngineering.ipynb` – предобработка и создание признаков.
- `03_Baseline_Models.ipynb` – сравнение логистической регрессии, Random Forest, LightGBM.
- `04_Neural_Network_MLP.ipynb` – нейросетевой базлайн.
- `05_Hyperparameter_Tuning_and_Calibration.ipynb` – подбор гиперпараметров Optuna, калибровка Platt scaling, логирование в MLflow.

### 4.2. Запуск сервиса (API/веб-интерфейс)

**Локально (без Docker):**

```bash
uvicorn src.app:app --reload --host 0.0.0.0 --port 8000
```

Или, если используется Docker:

```bash
docker build -t credit-scoring .
docker run -p 8000:8000 credit-scoring
```

- После запуска сервис доступен по адресу http://localhost:8000.

  - `GET/health` – проверка работоспособности.

## 6. Тесты (если есть)

Тесты находятся в папке `tests/`. Они включают:
- Модульные тесты (`test_model.py`):
  - загрузка артефактов;
  - sanity‑тест функции `predict` (проверка диапазона вероятности и типов);
  - проверка детерминированности.
- Smoke-тесты API (`test_api.py`):
  - проверка `GET/health`;
  - валидный запрос к `predict`;
  - невалидный запрос (отсутствие обязательных полей).

Пример:

```bash
cd project
pytest tests/ -v
```

## 8. Ограничения и дальнейшая работа

Текущие ограничения:
  - В сервисе используется только одна модель (LightGBM), нет A/B‑тестирования.
  - Метрики хранятся в памяти и сбрасываются при перезапуске.
  - Отсутствует авторизация и управление пользователями.

Возможные улучшения:
  - Добавить кэширование предсказаний для повторяющихся запросов.
  - Внедрить Prometheus + Grafana для более глубокого мониторинга.
  - Подключить распределённое логирование (ELK).
  - Реализовать автоматическое переобучение модели по расписанию (CI/CD).
