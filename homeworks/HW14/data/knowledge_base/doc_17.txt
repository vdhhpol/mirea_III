Оценка retrieval с использованием бенчмарка
Для объективной оценки качества retrieval необходимо создать бенчмарк — набор контрольных запросов с известными релевантными документами (ground truth). В учебном проекте бенчмарк можно создать вручную, выбрав запросы, которые однозначно соответствуют определённому документу из базы знаний. Рекомендуемый размер бенчмарка — 8–12 запросов. Каждому запросу должен соответствовать хотя бы один релевантный документ (или чанк). Важно, чтобы запросы покрывали разные аспекты тематики и имели разную сложность: точные совпадения ключевых слов, перефразирования, абстрактные вопросы. При оценке нужно зафиксировать параметры retrieval (top_k, модель эмбеддингов, чанкинг) и вычислить метрики hit@k, recall@k, MRR@k. Результаты удобно сохранить в CSV для анализа. Желательно также визуализировать метрики с помощью matplotlib. На основе бенчмарка можно сравнивать разные конфигурации (например, размер чанка) и выбирать лучшую. Бенчмарк должен оставаться неизменным при экспериментах, чтобы результаты были сопоставимы. В production‑системах бенчмарк обновляется по мере поступления реальных запросов и разметки от пользователей. Создание хорошего бенчмарка — это отдельная важная задача. Для учебных целей достаточно простого, но репрезентативного набора запросов.
