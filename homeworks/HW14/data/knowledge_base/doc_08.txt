TF‑IDF как альтернатива эмбеддингам
TF‑IDF (Term Frequency — Inverse Document Frequency) — это классический метод векторизации текстов, не использующий нейронные сети. Каждый текст представляется разреженным вектором, размерность которого равна размеру словаря. Значение каждого компонента — произведение частоты термина в документе (TF) на обратную частоту документа (IDF). TF‑IDF хорошо работает для точных совпадений ключевых слов, но не улавливает семантическую близость. Например, слова "автомобиль" и "машина" будут иметь разные координаты, хотя они синонимы. В отличие от dense эмбеддингов, TF‑IDF векторы разрежены, что позволяет использовать специальные индексы (например, inverted index) для быстрого поиска. В RAG‑системах TF‑IDF может использоваться как fallback на случай, если модель эмбеддингов недоступна. Также TF‑IDF часто комбинируют с dense эмбеддингами в гибридном поиске. Для вычисления TF‑IDF в Python используется sklearn.feature_extraction.text.TfidfVectorizer. После векторизации можно вычислять косинусное сходство между запросом и документами. Недостаток TF‑IDF — размер вектора может быть огромным (десятки тысяч), но разреженность позволяет эффективно хранить его. TF‑IDF не требует GPU и работает быстро на небольших корпусах. В учебных задачах TF‑IDF полезен для понимания основ векторизации. Однако для современных RAG‑систем dense эмбеддинги предпочтительнее из‑за лучшего семантического понимания. В некоторых предметных областях (например, юридические документы с точной терминологией) TF‑IDF может давать хорошие результаты.
