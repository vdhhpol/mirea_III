## 1. Dataset

- Какой датасет выбран: `S06-hw-dataset-01.csv`
- Размер: (12000, 30)
- Целевая переменная: `target` (бинарная классификация: класс 0 - 67,66%, класс 1 - 32,34%)
- Признаки: 25 числовых признака, 3 категориальных признака, идентификатор - 1 признак, целевая переменная - 1 признак.

## 2. Protocol

- Разбиение: train/test (75%/25%, `random_state=42`)
- Подбор: 5-фолдовая кросс-валидация на train, оптимизация ROC-AUC
- Метрики: accuracy, F1, ROC-AUC (Accuracy: базовая метрика для общей оценки; F1: учитывает дисбаланс классов (68%/32%); ROC-AUC: показывает качество разделения классов независимо от порога)

## 3. Models

Опишите, какие модели сравнивали и какие гиперпараметры подбирали.

Минимум:

- DummyClassifier (baseline): всегда предсказывает самый частый класс
- LogisticRegression (baseline из S05): `pipeline` со `StandardScaler`
- DecisionTreeClassifier:
    - Простое дерево без ограничений (глубина 24, 641 лист)
    - С подбором гиперпараметров: `criterion`, `max_depth`, `min_samples_leaf`, `min_samples_split`
    - Лучшие параметры: {`criterion`: `entropy`, `max_depth`: 7, `min_samples_leaf`: 4, `min_samples_split`: 10}
- RandomForestClassifier:
    - Простой случайный лес
    - С подбором гиперпараметров: `n_estimators`, `max_depth`, `max_features`, `min_samples_leaf`, `min_samples_split`
    - Лучшие параметры: `max_depth`: 20, `max_features`: `sqrt`, `min_samples_leaf`: 1, `min_samples_split`: 2, `n_estimators`: 200
- GradientBoosting:
    - Простой градиентный бустинг
    - С подбором гиперпараметров: `n_estimators`, `learning_rate`, `max_depth`, `min_samples_leaf`, `min_samples_split`
    - Лучшие параметры: `learning_rate`: 0.1, `max_depth`: 7, `min_samples_leaf`: 1, `min_samples_split`: 5, `n_estimators`: 200

## 6. Conclusion

- Градиентный бустинг и случайный лес значительно превосходят одиночные модели (логистическую регрессию и дерево решений) на данном датасете, демонстрируя лучшую способность к обобщению.
- Подбор гиперпараметров для деревьев решений (ограничение глубины, минимальное количество образцов в листьях) позволяет избежать переобучения и улучшить качество модели.
- В то время как простые модели (логистическая регрессия, неглубокие деревья) легче интерпретировать, сложные ансамблевые методы показывают значительно лучшие результаты предсказания.
- DummyClassifier служит важным ориентиром - любая осмысленная модель должна превосходить его по метрикам, учитывающим оба класса (F1, ROC-AUC).
